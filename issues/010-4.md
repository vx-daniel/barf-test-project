## Description
Implement the `toScientific(n, precision?)` number formatting utility that formats a number in scientific notation, and write its tests.

## Acceptance Criteria
- [ ] `toScientific(n, precision?)` formats a number in scientific notation (e.g. `1234` → `"1.234e+3"`)
- [ ] `precision` is optional; omitting it uses JS default of 6 significant digits via `toExponential()`
- [ ] Supports custom precision (e.g. `toScientific(1234, 2)` → `"1.23e+3"`)
- [ ] Returns `"0e+0"` for zero
- [ ] Handles negative numbers (e.g. `toScientific(-1234)` → `"-1.234e+3"`)
- [ ] Returns `"NaN"`, `"Infinity"`, `"-Infinity"` for those special values
- [ ] Exported from `src/index.ts`
- [ ] Tests pass in `tests/formatting.test.ts`

## Implementation Notes
- Guard NaN/±Infinity → return string literal
- Call `n.toExponential(precision)` — omit the argument entirely if `precision` is `undefined`
