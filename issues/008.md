## Description

## Acceptance Criteria
- [ ] Length conversions (meters, feet, inches, km, miles)
- [ ] Weight conversions (kg, lbs, oz, grams)
- [ ] Volume conversions (liters, gallons, cups, ml)
- [ ] Generic `convert(value, from, to)` API
- [ ] Tests for all conversion types

2. How should the API handle invalid conversions (e.g., converting kg to meters)?

3. What precision/rounding behavior is expected for conversion results?

   - Round to 2 decimal places

   - Configuration file or environment
   - Database/external source
