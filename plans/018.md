# Plan — 018: Fix verification failures for 010-4

## Context

Issue 010-4 (toScientific utility) was marked COMPLETED but failed automated verification.
The only failures are **Biome formatter violations** in two files — no logic changes needed.

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Use `bun run format --write` to auto-fix | Both violations are cosmetic line-length wrapping that Biome's formatter handles automatically — manual editing is error-prone and unnecessary |
| Verify all three acceptance criteria after the fix | The issue requires `bun run build`, `bun run check`, and `bun test` all passing |

## Failure Analysis

### File 1: `src/index.ts` line 116

The ternary expression in `toScientific` is too long for a single line:

```
return precision === undefined ? n.toExponential() : n.toExponential(precision)
```

Biome wants it wrapped as:

```typescript
return precision === undefined
  ? n.toExponential()
  : n.toExponential(precision)
```

### File 2: `tests/formatting.test.ts` line 50

The `.toThrow()` call with a string argument is too long for a single line:

```
expect(() => formatCurrency(10, 'ABC')).toThrow('Unknown currency code: ABC')
```

Biome wants it wrapped as:

```typescript
expect(() => formatCurrency(10, 'ABC')).toThrow(
  'Unknown currency code: ABC',
)
```
