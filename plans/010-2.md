# Plan: formatPercent utility

## Overview
Implement `formatPercent(n: number, decimals?: number): string` — a pure function that formats a ratio as a percentage string (e.g. `0.1234` → `"12.34%"`). Write comprehensive tests in `tests/formatting.test.ts`.

## Design Decisions

1. **Multiply-then-fix approach** — multiply input by 100, call `toFixed(decimals)`, append `"%"`. Simple, correct, and mirrors the acceptance criteria exactly.
2. **Default `decimals` to 2** — matches the most common percentage display convention. Use nullish coalescing (`decimals ?? 2`) so callers can explicitly pass `0`.
3. **NaN / ±Infinity guard-first** — return string literals (`"NaN%"`, `"Infinity%"`, `"-Infinity%"`) immediately without multiplication. These are valid IEEE 754 values and should not throw, consistent with the `formatWithCommas` pattern from issue 010-1.
4. **No `Intl.NumberFormat`** — keep the function pure and deterministic, consistent with the project convention of no side-effects/I/O and the sibling formatting utilities.
5. **Shared test file** — append to `tests/formatting.test.ts` (created by 010-1, or create it if it doesn't exist yet). Each formatting utility gets its own `describe` block.

```ts
// ── Percentage formatting (issue 010-2) ──────────────────────────────────────

/**
 * Formats a ratio as a percentage string (e.g. 0.1234 → "12.34%").
 * @param n - The ratio to format (0.5 = 50%)
 * @param decimals - Decimal places in the result (default: 2)
 */
export function formatPercent(n: number, decimals?: number): string {
  if (Number.isNaN(n)) return 'NaN%'
  if (!Number.isFinite(n)) return n > 0 ? 'Infinity%' : '-Infinity%'

  const d = decimals ?? 2
  return (n * 100).toFixed(d) + '%'
}
```

## Edge Cases & Risks

- **Floating-point precision**: `0.1 * 100` is exactly `10` in JS, but some ratios may exhibit floating-point drift (e.g. `0.1 + 0.2`). `toFixed()` handles rounding correctly for the specified decimal places, so this is safe for display purposes.
- **`-0`**: `formatPercent(-0)` → `(-0 * 100).toFixed(2)` → `"0.00"` → `"0.00%"`. The negative sign is dropped by `toFixed`, which is acceptable.
- **Very small decimals parameter**: `formatPercent(0.5, 0)` → `"50%"` (no decimal point). This is correct and tested.
- **Large decimals parameter**: `formatPercent(0.1, 20)` → `toFixed(20)` works up to 100 digits in most JS engines. No practical risk.
- **Values > 1.0**: `formatPercent(2.5)` → `"250.00%"`. This is mathematically correct and not restricted by the spec.
