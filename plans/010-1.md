# Plan: formatWithCommas utility

## Overview
Implement `formatWithCommas(n: number): string` — a pure function that adds thousand separators to a number and returns the formatted string. Write comprehensive tests in `tests/formatting.test.ts`.

## Design Decisions

1. **Hand-rolled implementation** — no `Intl.NumberFormat` or locale dependency. Keeps the function pure, deterministic, and consistent with the project convention of no side-effects/I/O.
2. **NaN / ±Infinity guard-first** — return the literal string representations (`"NaN"`, `"Infinity"`, `"-Infinity"`) immediately, matching `String(n)` behaviour. These are valid IEEE 754 values and should not throw.
3. **String splitting approach** — convert to string via `String(n)`, split on `"."`, handle negative sign separately, insert commas via regex or manual loop, rejoin. This avoids floating-point rounding issues since we never parse back to number.
4. **Regex for comma insertion** — use the standard `replace(/\B(?=(\d{3})+(?!\d))/g, ',')` pattern on the integer part. It's concise, well-understood, and correct for all integer lengths.
5. **Test file** — create `tests/formatting.test.ts`. This file will be shared by sibling issues (010-2 through 010-4) later, but for now contains only `formatWithCommas` tests.

```ts
/** Adds thousand separators to a number (e.g. 1234567 → "1,234,567"). */
export function formatWithCommas(n: number): string {
  if (Number.isNaN(n)) return 'NaN'
  if (!Number.isFinite(n)) return n > 0 ? 'Infinity' : '-Infinity'

  const str = String(n)
  const [intPart, decPart] = str.split('.')
  const isNegative = intPart.startsWith('-')
  const digits = isNegative ? intPart.slice(1) : intPart
  const withCommas = digits.replace(/\B(?=(\d{3})+(?!\d))/g, ',')

  let result = isNegative ? '-' + withCommas : withCommas
  if (decPart !== undefined) result += '.' + decPart
  return result
}
```

## Edge Cases & Risks

- **`1e20` and beyond**: `String(1e20)` returns `"100000000000000000000"` (20 zeros) which formats correctly. But `String(1e21)` returns `"1e+21"` which would not get commas — this is inherent to JS `String()` conversion. The acceptance criteria only require `1e15`, which is safe (15 zeros).
- **`-0`**: `String(-0)` returns `"0"`, so `formatWithCommas(-0)` returns `"0"` — acceptable.
- **Integer-only decimals**: `formatWithCommas(1234.0)` → JS coerces to `1234` → `"1,234"`. No decimal dot, which is correct.
