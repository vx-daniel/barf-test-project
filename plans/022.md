# Plan: 022 — Fix verification failures: 010-4

## Problem

Issue 010-4 (toScientific + formatCurrency formatting features) was marked
COMPLETED but failed the automated `bun run check` verification.
Two biome formatting violations were found — no logic bugs, no test failures.

### Failure 1 — `src/index.ts` line 128

The ternary in `toScientific` is too long for a single line per biome's rules:

```ts
// current (too long)
return precision === undefined ? n.toExponential() : n.toExponential(precision)

// expected (multi-line ternary)
return precision === undefined
  ? n.toExponential()
  : n.toExponential(precision)
```

### Failure 2 — `tests/formatting.test.ts` line 50

The `.toThrow()` call argument is too long for a single line:

```ts
// current (too long)
expect(() => formatCurrency(10, 'ABC')).toThrow('Unknown currency code: ABC')

// expected (wrapped)
expect(() => formatCurrency(10, 'ABC')).toThrow(
  'Unknown currency code: ABC',
)
```

## Design Decisions

| Decision | Rationale |
|----------|-----------|
