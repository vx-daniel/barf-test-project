# Plan: toScientific() notation formatter (issue 020)

## Overview

Implement `toScientific(n: number, precision?: number): string` that formats a number in scientific notation. Follows the existing pure-function pattern in `src/index.ts` — pure, validated, JSDoc-documented. This is a sibling of issues 017 (`formatWithCommas`) and 018 (`formatPercent`) under parent issue 010 (Number formatting).

## Design Decisions

1. **`toExponential` wrapper** — JavaScript's built-in `Number.prototype.toExponential(fractionDigits)` already produces the exact output format required: `(1234.56).toExponential(2)` → `'1.23e+3'`. Using it avoids reimplementing IEEE 754 exponent extraction and mantissa rounding.
2. **`precision` maps to `fractionDigits`** — The issue specifies "precision for mantissa (default: 2)" meaning digits after the decimal point in the significand. This maps directly to `toExponential`'s `fractionDigits` parameter.
3. **Default `precision = 2`** — Matches the issue requirement and example output (`1234.56` → `'1.23e+3'`).
4. **No input validation / no throws** — Consistent with sibling formatters (018's `formatPercent` takes the same approach). Invalid `precision` values (negative, non-integer) will propagate `toExponential`'s native `RangeError`, which is the correct behavior.

```ts
/**
 * Formats a number in scientific notation.
 * Optional `precision` controls the number of mantissa decimal digits (default: 2).
 * Examples: 1234.56 → '1.23e+3', 0 → '0.00e+0'
 */
export function toScientific(n: number, precision = 2): string {
  // Normalise -0 to 0 to avoid '-0.00e+0'
  const value = Object.is(n, -0) ? 0 : n
  return value.toExponential(precision)
}
```
