## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Guard NaN/±Infinity before calling `toExponential` | `toExponential(precision)` could behave unexpectedly with special values; explicit guards are clearer and match implementation notes |
| Omit `toExponential` argument when `precision` is `undefined` | JS default gives "as many digits as needed to uniquely specify the number" — matching the AC example `1234 → "1.234e+3"` |
| Single function in `src/index.ts`, tests in `tests/formatting.test.ts` | Follows existing project conventions (all exports from index.ts, test file name per AC) |

```typescript
// ── Formatting (issue 010-4) ────────────────────────────────────────────────

/** Formats a number in scientific notation. Omit precision to use JS default. */
export function toScientific(n: number, precision?: number): string {
  if (Number.isNaN(n)) return 'NaN'
  if (n === Infinity) return 'Infinity'
  if (n === -Infinity) return '-Infinity'
  return precision === undefined ? n.toExponential() : n.toExponential(precision)
}
```
