# Plan: Fix verification failures for 009-1

## Problem

Issue 009-1 (Standard Deviation) was implemented correctly — code and tests
pass — but automated verification failed because the sample-project's
`package.json` is missing `build` and `check` scripts:

```
error: Script not found "build"
error: Script not found "check"
```

Barf's verification system (`DEFAULT_VERIFY_CHECKS` in `src/core/verification.ts`)
runs three commands in sequence: `bun run build`, `bun run check`, `bun test`.
The first two are absent from the sample-project.

### Design Decisions

1. **`build` script — use `tsc --noEmit`**: The sample-project is a library
   consumed via direct imports, not a compiled binary. A type-check-only build
   (`tsc --noEmit`) validates TypeScript correctness without producing artefacts.
   This requires adding a `tsconfig.json`. We could alternatively use
   `bun build src/index.ts --outdir dist`, but since no consumer needs a `dist`
   folder, `tsc --noEmit` is lighter and more meaningful.

2. **`check` script — lightweight biome check scoped to this project**: The
   parent barf-ts `biome.json` explicitly excludes `**/sample-project` from its
   own checks. The sample-project should have its own minimal `biome.json` (or
   invoke biome directly) so `bun run check` validates formatting and linting
   locally. We'll add a local `biome.json` inheriting sensible defaults and a
   `check` script that runs `npx @biomejs/biome check ./src ./tests`.

3. **`tsconfig.json`**: Needed for `tsc --noEmit`. Minimal config with
   `strict: true`, targeting ESNext + Bun types, matching conventions in AGENTS.md.

4. **`typescript` devDependency**: `tsc` requires the `typescript` package.
   However, since we're in a Bun project that already has `@types/bun`, and
   `bun` ships its own type-checker via `bunx --bun tsc`, we can alternatively
   use `bunx --bun tsc --noEmit` which uses Bun's built-in TS support. To keep
   it simple and avoid adding dependencies, we'll use `bun build src/index.ts --outdir dist`
   for the build script instead — this validates that the code compiles and
   produces output without needing a separate TypeScript dependency.

   **Revised decision**: Use `bun build src/index.ts --outdir dist` for `build`.
   This validates compilation, is zero-config for Bun, and needs no extra deps.

## Risks

- **biome version**: The parent uses `@biomejs/biome@2.4.4`. Using `npx` will
  fetch/use whatever version is available. If the parent's `node_modules` is
  hoisted, it may resolve to 2.4.4 automatically. If not, npx will download it.
  Either way the schema version in `biome.json` matches.
- **dist/ in git**: The `bun build` output creates a `dist/` directory. We
  should ensure it's gitignored (or accept it as a build artefact). Adding
  `dist/` to `.gitignore` is prudent.
