# Plan 017 — Fix verification failures: 010-3

## Problem

Issue 010-3 (currency formatting) was marked COMPLETED but failed automated
verification. The `bun run check` step fails because Biome's formatter flags a
single line in `tests/formatting.test.ts` as too long:

```
tests/formatting.test.ts:50
  expect(() => formatCurrency(10, 'ABC')).toThrow('Unknown currency code: ABC')
```

Biome wants this line broken into a multi-line call:

```typescript
    expect(() => formatCurrency(10, 'ABC')).toThrow(
      'Unknown currency code: ABC',
    )
```

Current status of the three verification checks:
- `bun run build` — ✅ already passes
- `bun run check` — ❌ fails (1 format error in `tests/formatting.test.ts`)
- `bun test` — ✅ already passes (56 tests, 0 failures)

The lint step also passes (exit 0) — it emits one informational suggestion about
template literals in `src/index.ts:93` but this is a non-blocking `info` level
diagnostic, not an error.

## Design Decisions

1. **Fix only the formatting issue** — The sole blocking failure is a Biome
   format violation on one line. No source logic changes are needed.

2. **Do not apply the lint "info" suggestion** — The `useTemplate` suggestion in
   `src/index.ts:93` is informational (exit code 0) and doesn't block
   verification. Changing the return expression to a template literal would
   modify source code beyond what's necessary and risk altering behavior. Keep
   the fix minimal and scoped to the reported failure.

3. **Manual edit over `biome format --write`** — The fix is a single line. An
   explicit edit is more reviewable and avoids any risk of unintended formatting
   changes elsewhere.

## Implementation Steps

### Step 1: Fix formatting in `tests/formatting.test.ts`

**File:** `tests/formatting.test.ts`
**Action:** EDIT line 50

Replace:
```typescript
    expect(() => formatCurrency(10, 'ABC')).toThrow('Unknown currency code: ABC')
```

With:
```typescript

      'Unknown currency code: ABC',
    )
```

This breaks the long `.toThrow(...)` argument onto its own line with a trailing
comma, matching Biome's expected output exactly.

**Verify:** `bun run format` exits 0 (no format errors).
