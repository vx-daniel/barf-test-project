# Plan: Fix formatting violations in src/index.ts and tests/formatting.test.ts

## Overview
Issue 010-3 (formatCurrency) was completed but failed automated verification. The `bun run check` command (which runs `biome format`) reports two formatting errors — one in `src/index.ts` and one in `tests/formatting.test.ts`. Both are line-length issues where Biome wants to break long ternary/call expressions across multiple lines.

This is a verify-fix issue: no logic changes, just whitespace/formatting adjustments to satisfy the Biome formatter.

## Failures Analysis

### Failure 1 — `src/index.ts` line 116
The ternary on line 116 is too long for Biome's print width. Biome wants:
```ts
  return precision === undefined
    ? n.toExponential()
    : n.toExponential(precision)
```
Instead of the current single-line form.

### Failure 2 — `tests/formatting.test.ts` line 50
The `.toThrow(...)` call on line 50 is too long. Biome wants:
```ts
    expect(() => formatCurrency(10, 'ABC')).toThrow(
      'Unknown currency code: ABC',
    )
```
Instead of the current single-line form.

## Design Decisions

1. **Apply exact formatting Biome expects** — use `bun x biome format --write src/ tests/` to auto-fix, or manually apply the diff shown in the error output. Auto-fix is preferred since it's authoritative.
2. **No logic changes** — this is purely a formatting fix. Tests should continue to pass as-is.
