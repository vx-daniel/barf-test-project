# Plan 021 — Fix verification failures: 010-3

## Problem

Issue 010-3 (currency formatting) was marked COMPLETED but failed automated
verification on the `check` gate. Biome's formatter flags two lines that exceed
its preferred line-break threshold:

| # | File | Line | Issue |
|---|------|------|-------|
| 1 | `src/index.ts` | 128 | Ternary in `toScientific` too long — biome wants it split across 3 lines |
| 2 | `tests/formatting.test.ts` | 50 | `.toThrow(...)` call too long — biome wants the argument wrapped |

`bun run build` and `bun test` already pass (71 tests, 0 failures). Only
`bun run check` (biome format + lint) fails.

## Root Cause

Biome's default `printWidth` (80) causes it to reformat long expressions onto
multiple lines. The two lines above exceed that width and were committed
without running the formatter.

## Design Decisions

- **No config changes** — the biome config and print-width are correct; the
  source lines just need reformatting.
- **Manual edits preferred over `biome format --write`** — applying the exact
  diff biome shows keeps the change minimal and reviewable.

## Implementation Steps

### Step 1 — Fix `src/index.ts` line 128

Replace the single-line ternary:

```ts
  return precision === undefined ? n.toExponential() : n.toExponential(precision)
```

with the multi-line form biome expects:

```ts
  return precision === undefined
    ? n.toExponential()
    : n.toExponential(precision)
```

### Step 2 — Fix `tests/formatting.test.ts` line 50

Replace the single-line `.toThrow(...)`:

```ts
    expect(() => formatCurrency(10, 'ABC')).toThrow('Unknown currency code: ABC')
```

with the wrapped form biome expects:

```ts
    expect(() => formatCurrency(10, 'ABC')).toThrow(
      'Unknown currency code: ABC',
    )
```

### Step 3 — Verify all acceptance criteria

Run all three gates and confirm zero failures:

```bash
bun run build   # tsc --noEmit
bun run check   # biome format + lint
bun test        # 71 tests
```
