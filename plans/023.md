# Plan — Issue 023: Fix verification failures for 011

## Problem

Issue 011 (rounding utilities) was marked COMPLETED but failed the `bun run check` verification step. The sole failure is a **biome format** violation in `tests/rounding.test.ts`. No build or test failures — only formatting.

## Root Cause

Two categories of formatting violations:

1. **Line length**: 11 test lines exceed biome's default 80-character print width. Biome wants to break each long `it(...)` call so the `expect(...)` expression wraps to the next line with a 4-space indent continuation.
2. **Quote style**: Line 37 uses an escaped single quote inside a single-quoted string (`'banker\'s rounding'`). Biome prefers switching to double quotes when the string contains a literal single quote (`"banker's rounding"`), even though the project default is `quoteStyle: "single"`.

No source code (`src/`) changes are needed — only `tests/rounding.test.ts`.

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Fix method | Run `bun x biome format --write tests/rounding.test.ts` | Auto-formatter is authoritative; manual edits risk drift |
| Scope | Single file only | Formatter diff only flags `tests/rounding.test.ts` |
