# Plan: Range and Clamp

## Overview
Implement three numeric utilities: `range(start, end, step?)` for generating number arrays, `clamp(value, min, max)` for constraining a value within bounds, and `inRange(value, min, max)` for bounds checking. All are pure functions operating on plain numbers.

## Design Decisions

1. **`range` semantics** — generate numbers from `start` up to but **not including** `end`, matching Python's `range()` and lodash convention. This avoids off-by-one confusion and is the most widely expected behavior.
2. **`range` step inference** — when `step` is omitted, default to `1` if `start < end`, `-1` if `start > end`. A step of `0` always throws. If the step direction conflicts with start→end direction (e.g. `range(0, 5, -1)`), return an empty array rather than throwing — this matches lodash behavior and is safer.
3. **`clamp` simplicity** — `clamp(value, min, max)` returns `min` when `value < min`, `max` when `value > max`, otherwise `value`. Throws if `min > max` because that indicates a caller bug.
4. **`inRange` is fully inclusive** — `inRange(value, min, max)` returns `true` when `min <= value <= max`. Fully inclusive bounds are the most intuitive for a math utility (unlike lodash's half-open convention). Throws if `min > max`.

## Function Signatures

```ts
/** Generates an array of numbers from start (inclusive) to end (exclusive). */
export function range(start: number, end: number, step?: number): number[]

/** Constrains value to the inclusive range [min, max]. Throws if min > max. */
export function clamp(value: number, min: number, max: number): number

/** Returns true if value is within [min, max] (inclusive). Throws if min > max. */
export function inRange(value: number, min: number, max: number): boolean
```

### Step-by-step (TDD)

**Step 1 — `range`**
1. Create `tests/range.test.ts` with `describe('range', ...)` tests:
   - ascending: `range(0, 5)` → `[0, 1, 2, 3, 4]`
   - descending: `range(5, 0)` → `[5, 4, 3, 2, 1]`
   - custom step: `range(0, 10, 2)` → `[0, 2, 4, 6, 8]`
   - custom negative step: `range(10, 0, -3)` → `[10, 7, 4, 1]`
   - same start and end: `range(3, 3)` → `[]`
   - step zero throws: `range(0, 5, 0)` → throws
   - conflicting step direction: `range(0, 5, -1)` → `[]`
   - fractional step: `range(0, 1, 0.25)` → `[0, 0.25, 0.5, 0.75]`
2. Run `bun test` → tests fail (function doesn't exist)
3. Implement `range` in `src/index.ts`:
   - Throw on step === 0
   - Infer step as `1` or `-1` when omitted
   - Loop from start toward end, pushing values, stopping before reaching end
   - Return empty array when step direction conflicts with start→end
4. Run `bun test` → tests pass

**Step 2 — `clamp`**
1. Add `describe('clamp', ...)` tests to `tests/range.test.ts`:
   - value within range: `clamp(5, 0, 10)` → `5`
   - value below min: `clamp(-5, 0, 10)` → `0`
   - value above max: `clamp(15, 0, 10)` → `10`
   - value equals min: `clamp(0, 0, 10)` → `0`
   - value equals max: `clamp(10, 0, 10)` → `10`
   - negative bounds: `clamp(0, -10, -5)` → `-5`
   - min > max throws: `clamp(5, 10, 0)` → throws
2. Run `bun test` → fail
3. Implement `clamp` in `src/index.ts`:
   - Throw if `min > max`
   - Return `Math.max(min, Math.min(max, value))`
4. Run `bun test` → pass

**Step 3 — `inRange`**
1. Add `describe('inRange', ...)` tests to `tests/range.test.ts`:
   - value inside: `inRange(5, 0, 10)` → `true`
   - value outside below: `inRange(-1, 0, 10)` → `false`
   - value outside above: `inRange(11, 0, 10)` → `false`
   - value at min boundary: `inRange(0, 0, 10)` → `true`
   - value at max boundary: `inRange(10, 0, 10)` → `true`
   - negative range: `inRange(-5, -10, -1)` → `true`
   - min > max throws: `inRange(5, 10, 0)` → throws
2. Run `bun test` → fail
3. Implement `inRange` in `src/index.ts`:
   - Throw if `min > max`
   - Return `value >= min && value <= max`
4. Run `bun test` → pass

**Step 4 — Final verification**
1. Run `bun test` — all existing + new tests green
2. Verify all three functions are exported from `src/index.ts`

## Edge Cases

- **`range` with step 0** → throws `"Step cannot be zero"`
- **`range` with equal start/end** → returns `[]`
- **`range` with conflicting step** → returns `[]` (e.g. `range(0, 5, -1)`)
- **`range` with fractional step** → works correctly (e.g. `range(0, 1, 0.25)` → `[0, 0.25, 0.5, 0.75]`)
- **`range` descending** → auto-infers step of -1 when omitted
- **`clamp` / `inRange` with min > max** → throws `"min must not be greater than max"`
- **`clamp` at boundaries** → returns the boundary value (inclusive)
- **`inRange` at boundaries** → returns `true` (fully inclusive)
- **Negative numbers** → all three functions work with negative values
