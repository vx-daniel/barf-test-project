# Plan: isPrime Checker

## Overview
Implement an `isPrime(n)` function that determines whether a given number is prime, using trial division up to √n for efficiency.

## Design Decisions
- **Algorithm**: Trial division up to `Math.sqrt(n)`. Simple, correct, and efficient enough for all practical integer inputs. No need for probabilistic tests (Miller-Rabin) for a math utility library.
- **Return type**: `boolean` — consistent with the "checker" pattern (no throwing for non-prime inputs).
- **Edge cases**: 0, 1, and negative numbers all return `false` (they are not prime by definition). No need to throw — returning false is the mathematically correct answer.
- **Non-integer inputs**: `n <= 1` catches negatives and 0/1. For floats like 2.5, trial division naturally returns false since no integer cleanly divides them, but we can add an explicit `!Number.isInteger(n)` guard for clarity.

## Acceptance Criteria → Test Mapping

| Acceptance Criterion | Test Case(s) |
|---|---|
| Returns true for prime numbers | `isPrime(2)`, `isPrime(3)`, `isPrime(13)`, `isPrime(97)`, large prime `isPrime(7919)` |
| Returns false for composites | `isPrime(4)`, `isPrime(9)`, `isPrime(100)` |
| Handles edge cases: 0, 1, negative | `isPrime(0)`, `isPrime(1)`, `isPrime(-5)` |
| Efficient for large numbers (√n) | `isPrime(7919)` (large prime), `isPrime(10007)` — correctness implies the algorithm works; no perf benchmark needed |
| Tests cover primes, composites, edges | All of the above organized in describe blocks |

```ts
export function isPrime(n: number): boolean {
  if (!Number.isInteger(n) || n < 2) return false
  if (n === 2) return true
  if (n % 2 === 0) return false
  for (let i = 3; i * i <= n; i += 2) {
    if (n % i === 0) return false
  }
  return true
}
```
Key details:
- `i * i <= n` avoids `Math.sqrt` call each iteration (micro-optimization, also cleaner)
- Skip even divisors after checking 2
- `!Number.isInteger(n)` guards against floats
