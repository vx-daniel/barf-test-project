# Plan 015 — Fix verification failures: 009-2

## Problem

Issue 009-2 (variance calculation) was marked COMPLETED but failed automated
verification because the sample project is missing two required package.json
scripts:

| Check | Error |
|-------|-------|
| `bun run build` | `error: Script not found "build"` |
| `bun run check` | `error: Script not found "check"` |

barf's default verification runs three checks (from
`barf-ts/src/core/verification.ts`):

```typescript
{ name: 'build', command: 'bun', args: ['run', 'build'] }
{ name: 'check', command: 'bun', args: ['run', 'check'] }
{ name: 'test',  command: 'bun', args: ['test'] }
```

The `test` check already passes (39 tests across 5 files). Only `build` and
`check` are missing.

> **Note:** This is the same root cause as issue 014 (verify fix for 009-1).
> Plan 014 documented the approach in detail; this plan reuses that design
> since the fix was never applied.

## Design Decisions

1. **`build` = type-check with `tsc --noEmit`** — This is a library consumed as
   raw TypeScript by Bun. No compilation artifact is needed, so a type-check is
   the appropriate build step. Matches the AGENTS.md convention of "TypeScript
   strict mode."

2. **`check` = Biome format + lint** — Follows the same pattern as the parent
   barf-ts project (`"check": "bun run format && bun run lint"`) using Biome for
   both formatting and linting.

3. **Biome config mirrors barf-ts parent** — Single quotes, no semicolons,
   2-space indent, recommended lint rules. This matches the existing code style
   already used throughout the sample project.

4. **Minimal footprint** — Only two new config files (`tsconfig.json`,
   `biome.json`) and two new devDependencies (`typescript`, `@biomejs/biome`).
   No source code changes expected.

## Implementation Steps

### Step 1: Add `tsconfig.json`

Create `tsconfig.json` in the project root:

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "types": ["bun-types"],
    "noEmit": true
  },
  "include": ["src/**/*", "tests/**/*"]
}
```

Key choices:
- `strict: true` — matches AGENTS.md convention
- `types: ["bun-types"]` — resolves `bun:test` imports in test files
- `noEmit: true` — this project never emits JS, only type-checks
- `moduleResolution: "bundler"` — correct for Bun
- Mirrors the parent barf-ts tsconfig (minus the `paths` aliases which aren't
  needed here)

**Verify:** `bunx tsc --noEmit` exits 0.

### Step 2: Add `biome.json`

Create `biome.json` in the project root:

```json
{
  "$schema": "https://biomejs.dev/schemas/2.4.4/schema.json",
  "formatter": {
    "enabled": true,
    "indentStyle": "space",
    "indentWidth": 2
  },
  "javascript": {
    "formatter": {
      "quoteStyle": "single",
      "semicolons": "asNeeded"
    }
  },
  "linter": {
    "enabled": true,
    "rules": {
      "recommended": true
    }
  },
  "vcs": {
    "clientKind": "git",
    "defaultBranch": "main",
    "enabled": true,
    "useIgnoreFile": true
  }
}
```

Key choices:
- `semicolons: "asNeeded"` — existing code uses no semicolons
- `quoteStyle: "single"` — existing code uses single quotes
- `recommended: true` — standard lint rules, same as parent project
- Simplified from parent (no `assist` block, no custom `files.includes` needed)

**Verify:** `bunx biome check src/ tests/` exits 0.

### Step 3: Update `package.json`

Add devDependencies and scripts:

```jsonc
{
  "scripts": {
    "test": "bun test",
    "build": "bunx tsc --noEmit",
    "format": "bunx biome format --check src/ tests/",
    "lint": "bunx biome lint src/ tests/",
    "check": "bun run format && bun run lint",
    // ... existing scripts unchanged
  },
  "devDependencies": {
    "@types/bun": "latest",
    "typescript": "^5.0.0",
    "@biomejs/biome": "2.4.4"
  }
}
```

Key choices:
- `bunx tsc` / `bunx biome` — resolves tools from devDependencies
- `check` chains `format && lint` — mirrors parent barf-ts pattern exactly
- Biome version `2.4.4` — matches parent barf-ts for consistency
- `format` and `lint` target `src/ tests/` — covers all project code

### Step 4: Install dependencies

```bash
bun install
```

### Step 5: Run build and fix any type errors

```bash
bun run build
```

If there are type errors, fix them in `src/index.ts` or test files. Based on
code review, none are expected — all functions have explicit type annotations.

### Step 6: Run check and fix any format/lint issues

```bash
bun run check
```

If there are format or lint issues, fix with:
```bash
bunx biome format --write src/ tests/
bunx biome lint --fix src/ tests/
```

Based on code review, the existing style (no semicolons, single quotes, 2-space
indent) matches the Biome config, so minimal fixes should be needed.

### Step 7: Run full verification suite

```bash
bun run build && bun run check && bun test
```

All three must exit 0. This matches barf's verification checks exactly.
