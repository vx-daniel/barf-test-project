# Plan: Number Formatting

## Overview
Implement four number formatting utilities: `formatWithCommas` (thousand separators), `formatPercent` (percentage strings), `formatCurrency` (currency strings), and `toScientific` (scientific notation). All are pure functions that accept a number and return a formatted string.

## Design Decisions

1. **No Intl/locale dependency** — hand-roll formatting logic to keep functions pure, predictable, and free of locale side-effects. The acceptance criteria mention "locale edge cases" which we interpret as testing that our formatters handle the *inputs* edge-case numbers produce (NaN, Infinity, negatives, very large/small values) rather than switching locale at runtime.
2. **Optional parameters use simple defaults** — `decimals` defaults to 2, `currency` defaults to `"USD"`, `precision` defaults to undefined (use JS default of 6 significant digits in `toExponential`).
3. **Currency support** — keep a small lookup of common currency symbols (`USD` → `$`, `EUR` → `€`, `GBP` → `£`, `JPY` → `¥`). Throw on unknown currency code.
4. **NaN / Infinity handling** — return the literal strings `"NaN"`, `"Infinity"`, `"-Infinity"` for those inputs rather than throwing, since they are valid IEEE 754 numbers.

## Function Signatures

```ts
/** Adds thousand separators to a number (e.g. 1234567 → "1,234,567"). */
export function formatWithCommas(n: number): string

/** Formats a ratio as a percentage string (e.g. 0.1234 → "12.34%"). */
export function formatPercent(n: number, decimals?: number): string

/** Formats a number as a currency string (e.g. 1234.5 → "$1,234.50"). */
export function formatCurrency(n: number, currency?: string): string

/** Formats a number in scientific notation (e.g. 1234 → "1.234e+3"). */
export function toScientific(n: number, precision?: number): string
```

## Edge Cases

- **NaN** → returns `"NaN"` for all formatters
- **±Infinity** → returns `"Infinity"` / `"-Infinity"`
- **Zero** → `formatWithCommas(0)` → `"0"`, `formatPercent(0)` → `"0.00%"`, `formatCurrency(0)` → `"$0.00"`, `toScientific(0)` → `"0e+0"`
- **Negative numbers** → commas: `"-1,234"`, percent: `"-12.34%"`, currency: `"-$1,234.00"`
- **Very large numbers** → `formatWithCommas(1e15)` → `"1,000,000,000,000,000"`
- **Decimals in formatWithCommas** → `formatWithCommas(1234.5678)` → `"1,234.5678"` (commas only in integer part)
- **formatPercent with 0 decimals** → `formatPercent(0.126, 0)` → `"13%"`
- **Unknown currency code** → `formatCurrency(10, 'XYZ')` → throws
- **JPY (zero-decimal currency)** → still formats with 2 decimals for simplicity (consistent API)
